---
title: "Authentication"
description: "How to authenticate requests to the Perch Lending Services API"
---

All requests to the Lending Services API must include an `X-API-Key` header. This key identifies your firm and authorizes the request.

```bash
X-API-Key: YOUR_API_KEY
```

<Warning>
Keep your API key out of client-side code and version control. Store it in your backend environment only. Contact [partnersupport@myperch.io](mailto:partnersupport@myperch.io) immediately if you suspect your key has been compromised.
</Warning>

API keys are provisioned by the Perch team. To request access, contact [partnersupport@myperch.io](mailto:partnersupport@myperch.io) with your business details and intended use case. You will receive a key scoped to either the stable or production environment, or both.

## Troubleshooting

| Error | Cause | Resolution |
|-------|-------|------------|
| `401 Unauthorized` | Missing or invalid API key | Confirm the `X-API-Key` header is present and correctly spelled |
| `403 Forbidden` | Key lacks required permissions | Contact support to verify your key's permissions |
| `429 Too Many Requests` | Rate limit exceeded | Back off and retry — see [Rate Limits](/lending-services/rate-limits) |
